# MuleSoft Agent Fabric — Agent Broker Script
# Runtime: MuleSoft Flex Gateway 4.x  |  LLM: OpenAI GPT-4o

agent DiscountApprovalBroker:
  description: "Orchestrates multi-agent discount approval pipeline"
  model:       "gpt-4o"
  temperature: 0.1

  tools:
    - sfdc_query       # MCP-Salesforce: read opportunity + account data
    - sfdc_update      # MCP-Salesforce: write approval decision back
    - a2a_agentforce   # A2A: invoke Agentforce sub-agents
    - a2a_claude       # A2A: invoke Claude Agent for policy reasoning

  on_trigger: "salesforce.opportunity.discount_request"

  steps:

    ## Step 1 — Hydrate context from Salesforce
    - id: fetch_context
      tool: sfdc_query
      input:
        soql: |
          SELECT Id, Name, Amount, Account.Name,
                 Account.AnnualRevenue, Account.NumberOfEmployees,
                 Account.CustomerTier__c, CloseDate,
                 (SELECT Id, Amount FROM OpportunityLineItems)
          FROM Opportunity
          WHERE Id = '{{trigger.opportunity_id}}'
      output: opportunity

    ## Step 2 — Retrieve historical discount data
    - id: fetch_history
      tool: sfdc_query
      input:
        soql: |
          SELECT AVG(Discount__c) avg_disc, MAX(Discount__c) max_disc,
                 COUNT(Id) deal_count
          FROM Opportunity
          WHERE AccountId = '{{opportunity.AccountId}}'
            AND StageName = 'Closed Won'
      output: history

    ## Step 3 — LLM reasoning: should we approve?
    - id: reason
      type: llm_reasoning
      prompt: |
        You are a revenue-operations AI making a discount approval decision.

        OPPORTUNITY:   {{opportunity | json}}
        HISTORY:       {{history | json}}
        REQUESTED:     {{trigger.discount_pct}}%
        POLICY_LIMIT:  15%

        Evaluate whether the discount is justified. Return JSON:
        {
          "approved": true|false,
          "confidence": 0-1,
          "justification": "...",
          "conditions": ["..."]  // any attached conditions
        }
      output: decision

    ## Step 4 — If uncertain, invoke Claude Agent for policy cross-check
    - id: policy_check
      condition: "decision.confidence < 0.75"
      tool: a2a_claude
      input:
        task: "verify_discount_policy"
        context: "{{decision | json}}"
      output: policy_verdict

    ## Step 5 — Write decision to Salesforce
    - id: write_result
      tool: sfdc_update
      input:
        object: "Opportunity"
        id: "{{trigger.opportunity_id}}"
        fields:
          Approval_Status__c:    "{{decision.approved ? 'Approved' : 'Rejected'}}"
          Approval_Notes__c:     "{{decision.justification}}"
          Approved_Discount__c:  "{{trigger.discount_pct}}"
          ApprovedDate__c:       "{{now()}}"
  1. Receive Trigger Event
    Salesforce Platform Event fires on discount request. Ingress validates auth token and rate-limits the caller.
  2. Context Hydration via MCP-Salesforce
    Two parallel SOQL queries: live opportunity data + 36-month account discount history.
  3. LLM Reasoning Loop (OpenAI GPT-4o)
    Structured prompt with policy guardrails. Returns a typed JSON decision object with confidence score.
  4. Conditional A2A Policy Check (Claude Agent)
    Only invoked when confidence < 75%. Claude performs a secondary policy-alignment check using internal compliance docs.
  5. Write Approval Decision to Salesforce
    MCP-Salesforce updates Opportunity fields and posts a Chatter message with full LLM justification.
  6. Egress — Notify Sales Rep
    Platform Event published back. Salesforce Flow triggers email + in-app notification to the rep.
